OpenClaw 使用指南

以下示例基于默认安装路径撰写。若你所在环境经过自定义编译或部署，请以实际命令和配置为准。

核心概念

  • 项目/工作区：OpenClaw 操作对象的逻辑集合，包含配置、输入与产物。
  • 任务/管道：由一系列步骤组成的执行单元，可串并行编排。
  • 插件/扩展：为核心能力提供可插拔组件，便于按需启用。
  • Artifact：执行产生的二进制/报告/模型等可追溯产物。

快速上手

# 初始化工作区（在空目录或现有仓库中）
openclaw init

# 拉取示例模板（如适用）
openclaw template pull basic

# 首次运行
openclaw run

# 查看产物
openclaw artifacts ls

常用命令

# 帮助与版本
openclaw --help
openclaw --version

# 项目管理
openclaw init                # 初始化
openclaw config set k v      # 设置配置项
openclaw config get k        # 查看配置

# 任务执行
openclaw run                 # 按默认管道执行
openclaw run -p pipeline.yml # 指定管道
openclaw run --dry-run       # 仅校验不执行

# 产物与日志
openclaw artifacts ls
openclaw logs --tail 100

配置指南

默认配置文件位于 /etc/openclaw/config.yaml（Linux）或用户目录下的 ~/.openclaw/config.yaml。

# config.yaml 示例（模板）
telemetry: true
concurrency: 4
cache:
  enabled: true
  dir: ~/.cache/openclaw
registries:
  - name: public
    url: https://registry.example.com

企业环境请关闭遥测、调整并发与缓存目录，并配置私有 Registry。

实战示例

# 自定义管道（pipeline.yml）
steps:
  - name: prepare
    run: openclaw prepare --input data/
  - name: build
    run: openclaw build --target x86_64
  - name: verify
    run: openclaw verify --level strict

# 执行
openclaw run -p pipeline.yml

更多场景示例（CI 集成、容器化、多节点）将持续在 论坛 与文档更新。

排错建议

  • 遇到权限问题：优先使用非 root 用户，必要时为数据目录单独设定权限。
  • 网络超时：检查代理与私有源配置，容器环境确认 DNS 与镜像可达。
  • 命令不可用：确认 PATH 与安装路径；Windows 请在新开终端生效环境变量。
  • 仍未解决：收集日志 openclaw logs --since 10m，在论坛附上环境信息与操作步骤。
