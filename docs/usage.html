OpenClaw 使用指南

以下示例基于默认安装路径撰写。若你所在环境经过自定义编译或部署，请以实际命令和配置为准。

核心概念

  • 项目/工作区：OpenClaw 操作对象的逻辑集合，包含配置、输入与产物。
  • 任务/管道：由一系列步骤组成的执行单元，可串并行编排。
  • 插件/扩展：为核心能力提供可插拔组件，便于按需启用。
  • Artifact：执行产生的二进制/报告/模型等可追溯产物。

快速上手

# 运行配置向导（安装页完成后）
openclaw onboard --install-daemon

# 检查网关状态并启动
openclaw gateway status
openclaw gateway --port 18789

连接聊天平台

配对你的第一个聊天界面：

# WhatsApp（二维码登录）
openclaw channels login

Telegram/Discord 等可使用向导配置 Token 或参考相应文档。

安全与配对

默认未知 DM 会返回简短代码，需批准后才处理消息：

# 查看待批准的配对请求（示例：whatsapp）
openclaw pairing list whatsapp

# 批准配对，将 <code> 替换为实际代码
openclaw pairing approve whatsapp <code>

常用命令

# 帮助与版本
openclaw --help
openclaw --version

# 网关与体检
openclaw gateway status
openclaw status
openclaw health
openclaw security audit --deep

配置指南

核心配置位于 ~/.openclaw/。重要路径：

  • 主配置：~/.openclaw/
  • OAuth 凭证：~/.openclaw/credentials/oauth.json
  • 认证配置：~/.openclaw/agents/<agentId>/agent/auth-profiles.json

发送测试消息

openclaw message send --target +15555550123 --message "Hello from OpenClaw"

若 openclaw health 显示未配置授权，请回到配置向导完成 OAuth/密钥设置。

排错建议

  • 遇到权限问题：优先使用非 root 用户，必要时为数据目录单独设定权限。
  • 网络超时：检查代理；在服务器环境先完成 OAuth 再复制凭证。
  • Windows：优先使用 WSL2 以获得更好兼容性与性能。
  • 仍未解决：收集日志 openclaw logs --since 10m，在论坛附上环境信息与操作步骤。
