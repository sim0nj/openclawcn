常见问题

安装后执行提醒“找不到 openclaw”

请确认二进制已加入 PATH：

# macOS/Linux
echo 'export PATH="/usr/local/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc

# Windows
在“系统属性 → 环境变量”中，将安装目录加入 Path。

如何配置网络代理与私有源

  • 全局代理：设置 HTTP_PROXY/HTTPS_PROXY/NO_PROXY 环境变量。
  • 容器环境：在 Docker Daemon 或 Kubernetes 节点配置代理与镜像加速。
  • 私有源：在 config.yaml 的 registries 增加内网地址与凭据。

如何搭建离线仓库

推荐使用 Nexus/Harbor/Artifactory 作为软件包与容器镜像的集中仓库：

  1. 在联网环境同步公共仓库到内网。
  2. 将 OpenClaw 二进制与依赖镜像统一入库。
  3. 在目标机器只允许访问内网仓库域名。

权限不足或文件锁定

避免以 root 运行，给数据目录赋予当前用户权限：

sudo chown -R $(whoami) /var/lib/openclaw
sudo chmod -R u+rwX /var/lib/openclaw

如何收集日志帮助定位

openclaw logs --since 30m > openclaw-logs.txt
openclaw doctor --verbose > openclaw-diagnose.txt

将日志与操作步骤、系统版本在 论坛 发帖附上。

升级是否兼容

遵循主次版本语义化约定：补丁版本兼容、次版本尽量兼容、主版本可能引入破坏性更新。升级前请备份数据并在预生产验证。
