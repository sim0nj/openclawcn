OpenClaw 安装指南

按官方“快速开始”整理，优先推荐一键脚本与配置向导；Docker/源码等方式作为可选补充。

系统要求

  • Node.js ≥ 22
  • 操作系统：macOS、Linux 或 Windows（推荐 WSL2）
  • 可选：Brave Search API（启用网络搜索功能）

一键安装

推荐使用官方脚本进行自动安装：

curl -fsSL https://openclaw.ai/install.sh | bash

脚本会检测并安装 Node.js 与依赖，免去手动配置。

配置向导

安装后运行配置向导，完成网关、认证与聊天平台接入：

openclaw onboard --install-daemon
  • 网关类型：本地或远程
  • AI 模型认证：OpenAI（OAuth/API Key）或 Anthropic（推荐 API Key，亦支持 claude setup-token）
  • 聊天平台：WhatsApp 二维码、Telegram/Discord Bot Token 等
  • 守护进程：macOS 使用 launchd，Linux/WSL2 使用 systemd

配置文件位置：

  • 主配置目录：~/.openclaw/
  • OAuth 凭证：~/.openclaw/credentials/oauth.json
  • 认证配置：~/.openclaw/agents/<agentId>/agent/auth-profiles.json

若在无显示器服务器部署，先在可用浏览器的机器完成 OAuth，再将 oauth.json 复制到服务器对应目录。

启动网关

网关负责处理所有消息：

openclaw gateway status
openclaw gateway --port 18789

验证与体检

openclaw status
openclaw health
openclaw security audit --deep

若命令不可用，请检查 PATH 并返回配置向导补全授权。

可选方案

以下为进阶/企业环境常见选项（如需）：

  • 容器化与编排：使用自建镜像与 Helm Chart
  • 源码构建与离线安装：在内网镜像与包仓库下完成

包管理器安装

若已在公共/私有仓库发布包，可按如下方式安装：

# macOS (Homebrew)
brew tap openclaw/tap
brew install openclaw

# Debian/Ubuntu (APT)
echo "deb [trusted=yes] https://repo.example.com/openclaw stable main" | sudo tee /etc/apt/sources.list.d/openclaw.list
sudo apt update && sudo apt install openclaw

# RHEL/CentOS (YUM/DNF)
sudo dnf config-manager --add-repo https://repo.example.com/openclaw/openclaw.repo
sudo dnf install openclaw

以上为模板仓库配置，请替换为真实仓库地址与 GPG Key。

源码构建

如需从源码构建（便于打补丁或内核定制）：

git clone https://github.com/your-org/openclaw.git
cd openclaw
# 选择你项目使用的语言/构建系统：
# - Go:       make build
# - Rust:     cargo build --release
# - Node:     pnpm install && pnpm build
# - Python:   pip install -r requirements.txt && python setup.py bdist

请以实际仓库与构建脚本为准，并遵循公司代码审核与依赖镜像策略。

离线安装

  1. 在有网环境打包二进制和依赖（容器镜像/包缓存）。
  2. 通过离线介质（U 盘/内网文件服务）传输到目标机器。
  3. 按 快速开始 或 包管理器 步骤安装；容器环境导入镜像：docker load -i openclaw.tar。

如需企业级离线仓库（APT/YUM/NuGet/Nexus/Harbor），请参考 FAQ 的“离线源搭建”。

如需 Kubernetes 部署或离线方案，可在 论坛 获取社区清单。

卸载与升级

# 升级
openclaw upgrade

# 卸载（保留数据目录请先备份）
openclaw stop
openclaw uninstall

生产环境升级前请在预生产验证并做好数据备份。
