OpenClaw 安装指南

说明：以下为通用安装路径模板，已覆盖 macOS、Linux、Windows 及容器场景。若你已有官方二进制或私有源，请直接替换对应下载地址。

系统要求

  • CPU：x86_64 或 arm64
  • 内存：最低 2 GiB，推荐 4 GiB+
  • 磁盘：空闲 500 MiB（不含数据目录）
  • 操作系统：macOS 12+、Ubuntu 20.04+/Debian 11+、Windows 10/11
  • 可选依赖：Git、Docker（用于容器化部署）、curl/wget、Node.js（如需前端工具）

快速开始（官方二进制）

从官方发布页下载对应平台的压缩包，解压并添加到 PATH：

# macOS / Linux
curl -L -o openclaw.tar.gz https://example.com/openclaw/releases/latest/openclaw-$(uname -s)-$(uname -m).tar.gz
tar -xzf openclaw.tar.gz
sudo install -m 0755 openclaw /usr/local/bin/openclaw

# Windows (PowerShell)
Invoke-WebRequest https://example.com/openclaw/releases/latest/openclaw-windows-x64.zip -OutFile openclaw.zip
Expand-Archive openclaw.zip -DestinationPath $Env:ProgramFiles\\OpenClaw
$Env:Path += ";$Env:ProgramFiles\\OpenClaw"

替换示例地址为实际发布 URL。若采用企业内网镜像，请按内网规范替换域名。

Docker 部署

适合快速试用或生产容器化环境：

# 拉取镜像（示例仓库）
docker pull registry.example.com/openclaw:latest

# 运行（映射配置和数据目录）
docker run -d --name openclaw \
  -p 8080:8080 \
  -v /etc/openclaw:/etc/openclaw \
  -v /var/lib/openclaw:/var/lib/openclaw \
  registry.example.com/openclaw:latest

如需 Kubernetes 部署，建议在 论坛 下载官方 Helm Chart 或参考社区清单。

包管理器安装

若已在公共/私有仓库发布包，可按如下方式安装：

# macOS (Homebrew)
brew tap openclaw/tap
brew install openclaw

# Debian/Ubuntu (APT)
echo "deb [trusted=yes] https://repo.example.com/openclaw stable main" | sudo tee /etc/apt/sources.list.d/openclaw.list
sudo apt update && sudo apt install openclaw

# RHEL/CentOS (YUM/DNF)
sudo dnf config-manager --add-repo https://repo.example.com/openclaw/openclaw.repo
sudo dnf install openclaw

以上为模板仓库配置，请替换为真实仓库地址与 GPG Key。

源码构建

如需从源码构建（便于打补丁或内核定制）：

git clone https://github.com/your-org/openclaw.git
cd openclaw
# 选择你项目使用的语言/构建系统：
# - Go:       make build
# - Rust:     cargo build --release
# - Node:     pnpm install && pnpm build
# - Python:   pip install -r requirements.txt && python setup.py bdist

请以实际仓库与构建脚本为准，并遵循公司代码审核与依赖镜像策略。

离线安装

  1. 在有网环境打包二进制和依赖（容器镜像/包缓存）。
  2. 通过离线介质（U 盘/内网文件服务）传输到目标机器。
  3. 按 快速开始 或 包管理器 步骤安装；容器环境导入镜像：docker load -i openclaw.tar。

如需企业级离线仓库（APT/YUM/NuGet/Nexus/Harbor），请参考 FAQ 的“离线源搭建”。

安装验证

# 查看版本
openclaw --version

# 运行自检（示例）
openclaw doctor

# 启动服务（如适用）
openclaw start
openclaw status

若命令不可用，请确认 PATH 是否包含安装目录，或进入 论坛 搜索已解答的问题。

卸载与升级

# 升级
openclaw upgrade

# 卸载（保留数据目录请先备份）
openclaw stop
openclaw uninstall

生产环境升级前建议先在预生产环境回归验证，并做好数据备份。
